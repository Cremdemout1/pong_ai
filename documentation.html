AI Documentation

1. Overview

This project features two AI-controlled paddles for a 3D ball simulation. Each AI uses a different approach to track and intercept the ball:

  • Reactive AI: Uses current ball state and paddle state to choose immediate movement.
  • Predictive AI: Uses extended ball trajectory information to predict where to move next.

The goal of this experiment was to compare the learning capabilities of two almost identical machine learning algorithms and observe how critical the right input is to a ML algorithm.

3. Detailed AI Descriptions

3.1 Reactive AI

  • Goal: Move paddle according to ball's current state.
  • Inputs: Ball position and velocity, paddle position, paddle speed/height.
  • Algorithm: Feedforward neural network with a single hidden layer.
  • Action Selection: Chooses the action with the highest probability from softmax output.
  • Strengths: Fast response, simple implementation.
  • Weaknesses: Cannot anticipate ball trajectory beyond current velocity.

3.2 Predictive AI

  • Goal: Predict where the paddle needs to be to intercept the ball.
  • Inputs: Ball position/velocity, paddle state, distance to paddle, time to walls.
  • Algorithm: Feedforward neural network with a hidden layer. Uses trajectory information and combines top two actions when softmax outputs are close.
  • Action Selection: Chooses either top probability action or combines two close top actions for diagonal movement.
  • Strengths: Anticipates ball movement, more accurate at intercepting.
  • Weaknesses: Slightly slower to react due to extended input processing.

4. Technology Used

  • TypeScript for AI logic and simulation integration
  • Feedforward Neural Networks with sigmoid activation
  • Softmax for output action probabilities
  • Stochastic Gradient Descent for training
